# Media shot list — dressing the repo for visitors

LinkedIn traffic lands on a text README while the product's best asset is visual. Capture these,
drop them in `docs/media/`, and the README Demo section (already stubbed) gets wired up.

## Capture setup (once)
- Clean desktop: dark or plain wallpaper, hide desktop icons (right-click > View), close busy windows.
- GIFs: **ScreenToGif** (free), 15 fps, record at 1080p then export resized to 960px wide,
  target **under 10 MB** or GitHub renders it poorly.
- Video with AUDIO: **Win+Alt+R** (Xbox Game Bar) captures screen + mic. MP4s can be drag-dropped
  into a GitHub issue/PR comment to get a hosted URL that embeds in the README.
- Before recording: fresh briefing on file, charge above 30% (the PWR readout is in shot).

## Epic trailer — storyboard (~30s, 16:9)
